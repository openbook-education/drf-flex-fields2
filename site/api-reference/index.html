    API Reference¶

    Public Package Exports¶

    rest_flex_fields2 ¶

    Public API for the rest_flex_fields2 package.

    Exports configuration constants, the main serializer mixin and model serializer, the convenience view set, and the request-inspection helpers is_expanded and is_included.

    FlexFieldsModelSerializer ¶

    Bases: FlexFieldsSerializerMixin, ModelSerializer

    Convenience serializer combining FlexFieldsSerializerMixin with ModelSerializer.

    Drop-in replacement for serializers.ModelSerializer that adds sparse-fieldset (fields, omit) and nested-expansion (expand) support out of the box.

    FlexFieldsModelViewSet ¶

    Bases: FlexFieldsMixin, ModelViewSet

    Convenience view set combining FlexFieldsMixin with ModelViewSet.

    Drop-in replacement for viewsets.ModelViewSet with list-action expansion control provided by FlexFieldsMixin.

    is_expanded ¶

    is_expanded(request, field)

    Return whether field is requested for expansion.

    Inspects the expand query parameter on request. Returns True when field appears in the comma-separated expand list, or when a wildcard value (e.g. * or ~all) is present.

    is_included ¶

    is_included(request, field)

    Return whether field should be included in the response.

    Returns False when the fields sparse-fieldset parameter is present and field is not listed, or when the omit parameter is present and field is listed. Returns True otherwise.

    Serializers¶

    rest_flex_fields2.serializers.FlexFieldsSerializerMixin ¶

    Bases: Serializer

    Mixin that adds sparse-fieldset and nested-expansion support to a serializer.

    Accepts the fields, omit, and expand keyword arguments (names are configurable via REST_FLEX_FIELDS settings) both as constructor kwargs and as query-string parameters on the current request. Query parameters are only read on the root serializer; nested serializers receive their options through constructor kwargs propagated by the parent.

    Declare expandable relations either on Meta.expandable_fields (preferred) or directly on expandable_fields for backwards compatibility.

    apply_flex_fields ¶

    apply_flex_fields(fields, flex_options)

    Apply sparse-fieldset and expansion options to fields in place.

    Removes fields that are excluded by omit or not present in fields (sparse-fieldset), then replaces fields listed in expand with their nested serializer instances. Returns the modified fields mapping.

    Views¶

    rest_flex_fields2.views.FlexFieldsMixin ¶

    View mixin that restricts which fields may be expanded on list actions.

    Set permit_list_expands to a list of field names that are allowed to be expanded when the view is handling a list request. The allowed names are passed to the serializer via context['permitted_expands'] so that FlexFieldsSerializerMixin can enforce the constraint.

    get_serializer_context ¶

    get_serializer_context()

    Extend the serializer context with permitted_expands for list actions.

    When the current action is list, adds context['permitted_expands'] populated from permit_list_expands so the serializer can restrict expansion accordingly.

    Filter Backends¶

    rest_flex_fields2.filter_backends.FlexFieldsDocsFilterBackend ¶

    Bases: BaseFilterBackend

    No-op filter backend that adds flex-fields parameters to the API schema.

    Does not modify the queryset. Its sole purpose is to expose the fields, omit, and expand query parameters in the OpenAPI schema generated by tools such as drf-spectacular.

    filter_queryset ¶

    filter_queryset(request, queryset, view)

    Return queryset unchanged.

    get_schema_operation_parameters ¶

    get_schema_operation_parameters(view)

    Return the OpenAPI query parameter definitions for the flex-fields params.

    Emits fields, omit, and expand parameter objects for the view's serializer when it is a FlexFieldsSerializerMixin subclass. Returns an empty list for views that do not use flex fields.

    rest_flex_fields2.filter_backends.FlexFieldsFilterBackend ¶

    Bases: FlexFieldsDocsFilterBackend

    Filter backend that optimises querysets based on the active flex-fields options.

    Extends FlexFieldsDocsFilterBackend with actual queryset manipulation: applies only() to restrict fetched columns and select_related() / prefetch_related() for expanded relation fields. Only active for GET requests on views whose serializer is a FlexFieldsSerializerMixin subclass.

    View-level opt-out attributes:

    • auto_remove_fields_from_query (default True): disable only() optimisation.
    • auto_select_related_on_query (default True): disable relation prefetching.
    • required_query_fields (default []): extra field names always included in the only() call.

    filter_queryset ¶

    filter_queryset(request, queryset, view)

    Apply field-selection and relation-prefetch optimisations to queryset.

    Resolves the active fields / omit / expand options from the request, then calls only(), select_related(), and prefetch_related() as appropriate. Returns queryset unchanged when the view's serializer is not a FlexFieldsSerializerMixin subclass, or when the request method is not GET.

    rest_flex_fields2.utils.split_levels ¶

    split_levels(fields)

    Split a dot-notation field list into current-level and next-level parts.

    Given an iterable such as ['a', 'a.b', 'a.d', 'c'], returns a tuple (first_level, next_level) where first_level is the deduplicated list of top-level names (e.g. ['a', 'c']) and next_level is a dict mapping each name to its remaining path fragments (e.g. {'a': ['b', 'd']}). A plain string is treated as a comma-separated field list.
